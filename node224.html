数值稳定性

在LAPACK [12]中可获得的稳健的QR算法实现，计算矩阵A的近似Schur形式，满足

(A+E)\hat{V} = \hat{V} \hat{R},
 (128)

其中\hat{R}是上三角矩阵，\Vert \hat{V}^\ast \hat{V} - I\Vert \approx \epsilon_M ，并且\Vert E\Vert \approx \epsilon_M\Vert A\Vert，\epsilon_M是机器精度（单位舍入误差）。由于在迭代过程中仅使用酉变换，\Vert E\Vert \approx \epsilon_M\Vert A\Vert这一事实直接导致了QR算法的后向稳定性。换句话说，QR方法计算的是接近A的矩阵A+E的Schur形式。

后向稳定性对于任何数值算法都是一个非常令人安心的特性，但重要的是要记住，后向稳定性本身并不意味着准确的答案。计算出的特征值和特征向量的准确性取决于A的特征系统的敏感性（条件数）。后向稳定的算法对于良态问题会产生准确的答案。读者可参考§7.13以获得关于此问题的更详细讨论。

如上一节所述，IRAM是截断的QR迭代。在大多数QR方法的实现中，使用Householder变换来获得初始的Hessenberg形式简化。Arnoldi过程，如果扩展到n步，则是获得这种简化的另一种方式。通过引入正交校正 [96]，Arnoldi过程可以产生与Householder简化相同数值质量的简化，但它更适合于大规模设置。通过这种校正，计算出的Arnoldi向量在机器精度范围内是酉的(\Vert V_k^*V_k - I_k \Vert \approx \epsilon_M)，并且由于重启机制涉及QR机制中使用的相同酉变换，IRAM也是后向稳定的。

在IRAM迭代的任何阶段，我们有

(A + E_o) V_k = V_k H_k + f_k e_k^*,

其中\Vert E_o\Vert \approx \Vert A\Vert\epsilon_M。如果我们成功地使\Vert f_k\Vert < \Vert A\Vert\epsilon_M，那么

(A + E) V_k = V_k H_k,

其中E = E_o + f_k (V_k e_k)^*。如果H_k Z_k = Z_k R_k是H_k的Schur分解，那么

(A + E) \hat{V}_k = \hat{V}_k R_k \ \ \mathrm{with} \ \ \hat{V}_k = V_k Z_k

是附近矩阵A+E的精确部分Schur分解。

更有可能的是，在某个阶段，H_m（m = k+p）的特征向量的最后一个分量很小，收敛将被实现。我们仍然有

(A + E_1) V_m = V_m H_m + f_m e_m^*,

其中E_1相对于\Vert A\Vert很小。如果H_m y = y \theta，其中\eta = e_m^*y很小（例如\Vert f_m\Vert \vert\eta\vert < \epsilon_U \Vert H_m \Vert ），那么 (A + E) x = x , 其中x = V_m y，E = E_1 + f_m (e_m^*y) x^*（假设\Vert y\Vert = 1）。因此，在任何情况下，收敛的近似特征对(x, \theta)对于附近矩阵A+E是精确的。

通过使用我们将在§7.6.6中介绍的收缩技术，可以利用酉变换来获得关于一组k个收敛特征值的部分Schur分解的后向稳定性声明。如果有k个H_m的特征值已经收敛，可以构造一个酉矩阵Q，使得Q^* H_m Q的领先k \times k子矩阵R_k是上三角的，并且

( A + \hat{E}) \hat{V}_k = \hat{V}_k R_k , \ \\mathrm{with} \ \ \hat{V}_k = V_m Q,

其中\hat{E}相对于\Vert A\Vert很小。关于这一点的讨论见§7.6.6，更多细节可参见 [420]。
